Things to avoid while replying/forwarding/posting new messages to group

by Humble Groups (thirumalaikv), on Tue, Oct 30 2007, 17:54 IST

All,

While testing the archive process and displaying the messages on
the archive site, I came across few things which creates new topic
where as it shouldn't and threads been hijacked to different the
threads.

Guideline 1

Please do not create new mail and change the subject to already
existing thread subject, this will create new topic in yahoo groups
instead of adding to existing thread.

Guideline 2

Please do not reply to last message posted/random message and
change the subject to already existing thread, yahoo group will attach
your message to the random message thread instead of the actual thread
subject.

If you need more info, please go ahead and rest of the long
technical detail.

Allow me to give technical background info about how yahoo groups
maintain the thread format of all the messages. Each email message has
a unique message id which is supposed to be unique across all emails
around the world.

For eg., if you look at this message I posted


http://groups.yahoo.com/...9872?source=1&var=1&l=1


the header contains the following information

Message-ID: .

Now how yahoo group knows that it should come under the message
posted by "SatishKumar Arunachalam"


http://groups.yahoo.com/...9869?source=1&var=1&l=1

.
?

If you check the message id of the second one and Message-ID:
and In-Reply-To:
of my message both are same. This is how yahoo message arrange the
thread format.

If there is no In-Reply-To header, then yahoo group assumes that it
is a new topic.

Issue 1:

Now, suddenly a message posted with same subject as above doesn't
come under the thread, why? I assume the poster would have created a
new email instead of replying to the above email and sent it. Yahoo
group assumes that you created a new topic though the subject is same.

Issue 2:

A message posted with different subject but appears under
different thread why?. In this case the poster would have just reply
the last message or some random message and change the subject the
above subject. This creates a long thread even though it doesn't have
related messages.

Regards
Humble
